# 最近改动手动验证清单

## 1. Xterm / 不支持内容提示

- 打开包含 xterm magic tab 的帖子详情。
- 预期：不再直接渲染 xterm 内容，显示“不支持内容”提示。
- 预期：终端报告正文、隐藏输入框内容和 xterm 样式文本都不出现在详情正文里。

## 2. NodeSeek Jump 链接

- 打开详情页里包含 `/jump?to=...` 的链接。
- 点击跳转链接。
- 预期：这类 redirect 链接走 Safari，不进入 App 内原生帖子详情。
- 普通 NodeSeek 帖子链接仍应进入 App 原生详情页。

## 3. 详情图片缩略图缓存

- 打开包含多张图片的帖子详情。
- 首次进入时观察图片加载。
- 返回后再次进入同一帖子。
- 预期：已加载过的缩略图展示更快，图片布局不抖动、不遮挡文字。
- 点开图片预览。
- 预期：预览仍能正常打开。

## 4. 侧边菜单交互

- 在列表页从屏幕左侧边缘轻扫或拖动。
- 预期：侧边菜单能通过左侧 bounce/手势打开。
- 点击顶部菜单按钮。
- 预期：菜单正常打开，并有轻微触感反馈。

## 5. 已访问帖子缓存

- 在列表页点击一个未访问帖子。
- 预期：被点击的这一行标题立即变灰，只更新当前 row，列表不整体闪烁、不回到顶部。
- 返回列表。
- 预期：该帖子仍保持灰色。
- 下拉刷新列表。
- 预期：同一个帖子如果仍在列表里，继续保持灰色。
- 切换分类或排序后再回来。
- 预期：访问过的同一帖子仍然显示灰色。
- 将 App 退到后台，等待约 1 秒后杀掉重启。
- 预期：访问过的帖子仍可根据本地缓存显示灰色。

## 6. 回归检查

- 列表首次加载、下拉刷新、分页加载更多都应正常。
- 进入帖子详情、返回列表、再次进入其他帖子都应正常。
- 登录入口、侧边菜单入口不应受影响。
- 如果看到列表项全量刷新、滚动位置突然跳动、或标题颜色异常，优先记录复现步骤。
