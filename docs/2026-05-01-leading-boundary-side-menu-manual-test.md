# 首页左边界拉出侧栏手动测试

这次改动只影响首页横向分页在最左侧的边界拖拽行为。重点确认系统分页、列表纵向滚动和侧栏打开没有互相干扰。

## 测试步骤

- [ ] 打开首页，确认当前选中的是“全部”列表。
- [ ] 在“全部”列表里向右横向拖拽一小段后松手，列表应只做边界回弹，不打开侧栏。
- [ ] 在“全部”列表里向右横向拖拽超过明显阈值后松手，回弹时应触发一次轻量震动，并打开左侧栏。
- [ ] 在“全部”列表里正常纵向滚动，不能误触发侧栏。
- [ ] 从“全部”横向滑到“日常”等非首个列表后，再向右横向拖拽，应该正常切回前一页或停留在分页逻辑内，不直接打开侧栏。
- [ ] 左侧栏打开后，点击遮罩关闭；重复拉出和关闭几次，侧栏状态应稳定。

## 备注

- 这次实现监听的是 `UIPageViewController` 内部横向 `UIScrollView` 现有的 `panGestureRecognizer`，没有新增额外手势，也没有替换内部 scroll view 的 delegate。
